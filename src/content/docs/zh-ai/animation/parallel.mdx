---
title: 并行动画
---

import { Aside } from "@astrojs/starlight/components"

并行动画主要分为 `pre_parallel` 和 `parallel` 两大类。其中，`pre_parallel` 的优先级低于主动画，会被主动画覆盖；而 `parallel` 的优先级高于主动画，**并且采用了特殊的混合动画方式**。

并行动画同样放置在 `main.animation.json` 文件中，用于制作那些无论在何种情况下都会播放的动画内容，例如尾巴和耳朵的摆动、眨眼动画等。

你还能在并行动画中添加 molang 脚本，以便在特定情况下显示或隐藏护甲。

| 名称                                                            | 作用                                                                                 | 备注     |
| :-------------------------------------------------------------- | :----------------------------------------------------------------------------------- | :------- |
| `pre_parallel0`<br/>`pre_parallel1`<br/>...<br/>`pre_parallel7` | 与主动画相互独立，优先级最低。不同 `pre_parallel` 动画的优先度按照数字大小依次增高。 | 循环播放 |
| `parallel0`<br/>`parallel1`<br/>...<br/>`parallel7`             | 与主动画相互独立，优先级最高。不同 `parallel` 动画的优先度按照数字大小依次增高。     | 循环播放 |

<Aside title="什么是特殊的混合动画？">
假设存在两个动画：A 动画使手臂弯曲 10 度，B 动画使手臂弯曲 25 度，且 B 动画的优先级高于 A 动画。
- 默认情况：B 动画会完全覆盖 A 动画对手臂的影响，最终手臂弯曲 25 度。
- 特殊混合动画：B 动画和 A 动画对手臂的弯曲效果会相互叠加，最终手臂弯曲 35 度。

这种混合方式仅针对旋转进行叠加，不会对位移和缩放产生影响。

</Aside>
